## 윈도우 프로그램
* Windows 운영체제에서 실행 할 수 있는 응용 프로그램
* 특징
  * 장치 독립적
  * 멀티태스킹을 지원하여 여러 윈도우 프로그램이 동시에 실행됨
  * 메시지 구동방식(Message-Driven)으로 동작
  * 사용자 인터페이스인 GUI(Graphic User Interface)로 동작

## API(Application Programming Interface)
* 운영체제가 응용 프로그램을 위해 제공하는 함수의 집합
* Windows 운영체제가 제공하는 응용 프로그램을 위한 함수 집함은 WinAPI라고 함

## 관련 기초 용어
* 운영체제<br>
  * 컴퓨터의 하드웨어를 직접적으로 관리하는 소프트웨어
* 커널<br>
  * 운영체제의 핵심부분으로 운영체제의 다른 부분 및 응용 프로그램 수행에 필요한 여러가지 서비스 제공
* 멀티태스킹
  * 다수의 작업(혹은 프로세스, 이하 태스크)이 CPU와 같은 공용자원을 나누어 사용하는 것
  * 스케줄링이라는 방식을 사용하여 컴퓨터 사용자에게 병렬 연산이 이루어지는 것과 같은 환경 제공
* 이벤트(Event)
  * 윈도우 프로그램은 사용자로부터 마우스 클릭, 문자 입력 등의 이벤트에 의하여 프로그램이 진행
* 메시지(Message)
 * 윈도우 운영체제는 이벤트가 발생하면 메시지를 발생하여 해당 응용프로그램으로 보냄
* 프로젝트
  * 하나의 프로그램을 만드는 작업으로 특정 프로그램을 만들기 위해서 반드시 프로젝트를 구성 필요
* 솔루션
  * 프로젝트들의 모임
* 윈도우 프로시저(Callback 함수)
  * 응용프로그램이 제공하며 운영체제가 필요할 때 호출하는 함수로 “운영체제에 의해 호출되는 프로그램 내부의 함수”
* 시스템 메시지 큐
  * 운영체제가 관리하는 것으로 각 스레드 별로 하나씩 메시지 큐를 생성
  * 메시지 큐는 들어온 순서대로 메시지를 쌓아놓음
  * 시스템 메시지 큐는 시스템 전체에 유일한 메시지 큐이며 모든 큐 메시지는 먼저 이 곳에 저장
  * 시스템은 큐의 메시지를 하나씩 꺼내어 어떤 스레드로 보낼 메시지인지 판단하여 스레드 메시지 큐로 메시지를 보내고 시스템 메시지 큐에서 메시지를 지움
  * 그러면 메시지 루프에 의해 이 메시지는 해당 윈도우의 윈도우 프로시저로 보내져 처리되고 스레드 메시지 큐에서 메시지는 삭제
  * 큐(Queue)
    * FIFO의 원칙에 따라 운영되는 자료구조
