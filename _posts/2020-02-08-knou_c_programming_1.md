## C 언어의 정의
* C언어<br>
  프로그래밍 언어로 고급 언어지만 하드웨어 제어가 가능(포인터를 이용한 메모리 접근)하여 저급 언어의 특성을 지님
  * 저급 언어(high-level)<br>
  인간이 이해하기 쉬운 언어
  * 고급 언어(low_level)<br>
  컴퓨터가 이해하기 쉬운 언어 (어셈블리어)
* 기계어<br>
  0, 1로 구성된 컴퓨터가 사용하는 언어
* 컴파일러<br>
  프로그래밍 언어로 작성된 프로그램을 컴퓨터가 이해할 수 있도록 기계어로 번역해주는 번역기<br>
  소스 프로그램을 한번에 기계어로 변환시킴<br>
  프로그램을 구성하는 문자들을 기본 처리 단위인 토큰으로 분리하고 그것이 올바른 규칙을 따르는지 검사하여 실행코드를 생성
  * 어셈블러(assembler)<br>
    기호로 표현된 어셈블리 코드를 기계어로 번역하는 번역기
  * 인터프리터(interpreter)<br>
    프로그램을 한 단계씩 기계어로 해석해여 실행하는 언어처리 프로그램
* 사람 - 컴파일러 - 컴퓨터
  해당 순서로 상호 소통
  
## C 언어의 특징
* 프로그램 이식성이 높음
* 저급 언어의 특성을 가진 고급 언어

## C 프로그램의 완성 과정
1. 코딩 단계<br>
  소스코드를 작성하여 소스파일을 생성하는 과정
2. 컴파일 단계<br>
  소스파일이 목적 파일로 변환되는 과정
    * 목적 파일(object file)
      * .obj라는 확장자를 갖는 파일
      * 컴퓨터가 이해할 수 있는 기계어들의 집합으로 변경된 상태
      * 목적 파일은 그 자체로는 큰 의미를 갖지 못하고 라이브러리라는 파일과 연결되어 하나의 프로그램이 되었을 때에만 의미를 가짐
3. 링킹 단계<br>
  목적 파일을 실행 파일로 변환하는 과정으로 컴파일된 목적 파일들을 라이브러리 파일과 연결하여 실행 파일을 만들어 냄
  
## C 프로그램의 구성
```
// 도입부: 프로그램 전체에 적용되는 사항을 기술
# include <stdio.h>
int add(int x, int y);

// main() 함수: 가장 먼저 호출되는 함수로써 반드시 존재
void main() {
  // ...
}

// 호출된 함수: main() 함수에서 호출된 함수를 정의하는 부분 (도입부와 main() 함수 사이에 있어도 상관 없음)
int add(int x, int y) {
  return x + y;
}
```
#### C 프로그램의 기본 구조
* C 프로그램은 반드시 하나 이상의 함수를 포함해야 함
* main() 함수가 반드시 존재해야 함
* 함수의 시작과 끝을 알리는 중괄호({})를 사용해야 함
* 중괄호 안에는 변수선언문, 치환문, 연산문, 함수 등의 명령을 기입
* 선행처리기 부분을 제외하고는 문장의 끝에는 세미콜론(;)을 붙임
  * 선행처리기(preprocessor)
#### C 프로그램의 구성 요소
* 예약어(reserved word)<br>
  미리 정의되어 있는 의미 있는 단어
  * 자료형 관련 예약어<br>
    char, int, float, short, long, double, unsigned, union, enum, void
  * 기억 관련 예약어<br>
    auto, static, extern, register
  * 제어 관련 예약어<br>
    if~else, for, while, do~while, switch~case, break, continue, return
  * 기타 예약어<br>
    main, sizeof, include, ...
* 명칭<br>
  변수, 배열, 함수 등의 이름
* 상수<br>   
  값이 불변인 자료
* 연산자<br>
  =, -, *, /, ++ ...
* 주석
  * 여러 줄에 걸친 주석은 /* 와 */ 사용
  * 한 줄짜리 주석은 // 사용
